Actions are the core building blocks of ServFlow workflows. They perform discrete operations such as fetching data from a database, calling external APIs, running AI agents, or transforming values. Every workflow consists of an entry point, one or more actions, and a response.

You can configure actions in two ways:

- **Dashboard UI** â€” Visual interface for building workflows by selecting action types and filling in configuration fields. Best for learning, prototyping, and rapid development.
- **YAML Configuration** â€” Code-based configuration files for version control, CI/CD pipelines, and production deployments.

Both approaches produce the same result â€” the dashboard saves configurations as YAML files, so you can seamlessly switch between them. See [Running Modes](/running-modes) to understand when to use each approach.

---

## Action Structure

Every action follows this structure in YAML:

```yaml
actions:
  action_id:
    type: action_type
    config:
      # type-specific configuration
    next: action.next_step
    fail: response.error
```

The `action_id` is a unique identifier you choose for the action. You'll use this ID to reference the action's results in other parts of your workflow.

### Common Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | The action type identifier (e.g., `fetch`, `http`, `agent`) |
| `config` | object | No | Type-specific configuration options |
| `next` | string | No | Step to execute on success |
| `fail` | string | No | Step to execute on failure |

---

## Chaining Actions

Use the `next` and `fail` fields to define your workflow's execution path:

- **`next`** â€” Routes to another step when the action succeeds
- **`fail`** â€” Routes to an error handler when the action fails

The routing syntax follows the pattern `type.identifier`:

- `action.other_action` â€” Route to another action
- `conditional.check_value` â€” Route to a conditional branch
- `response.success` â€” Route to a response

## Accessing Action Results

When an action completes, its result is stored and can be referenced by subsequent nodes. Use the `{{ .action_id }}` syntax to access these values.

```yaml
# Access the full result from action 'fetch_user'
value: "{{ .fetch_user }}"

# Access a nested field
value: "{{ .fetch_user.email }}"

# Use in expressions
expression: "{{ gt (len .fetch_user) 0 }}"
```

For example, if an action with ID `get_user` returns user data, you can access the email field in a later action with `{{ .get_user.email }}`.

## Filter Syntax

Several actions use filters to query data. Filters follow this structure:

```yaml
filters:
  - field: field_name
    operator: eq
    value: "comparison_value"
```

### Available Operators

| Operator | Description |
|----------|-------------|
| `eq` | Equal to |
| `ne` | Not equal to |
| `gt` | Greater than |
| `gte` | Greater than or equal |
| `lt` | Less than |
| `lte` | Less than or equal |
| `in` | Value in array |
| `nin` | Value not in array |

**Example with multiple filters:**

```yaml
filters:
  - field: status
    operator: eq
    value: active
  - field: age
    operator: gte
    value: "18"
  - field: role
    operator: in
    value:
      - admin
      - moderator
```

## Available Actions

ServFlow provides actions organized by category:

### Data Operations
| Action | Description |
|--------|-------------|
| [`fetch`](/actions/data-operations#fetch) | Retrieves data from a database table |
| [`store`](/actions/data-operations#store) | Stores a new record in a database table |
| [`update`](/actions/data-operations#update) | Updates existing records |
| [`delete`](/actions/data-operations#delete) | Deletes records from a database table |
| [`mongoquery`](/actions/data-operations#mongoquery) | Executes native MongoDB queries |

### Vector Operations
| Action | Description |
|--------|-------------|
| [`storevector`](/actions/vector-operations#storevector) | Stores vector embeddings |
| [`fetchvectors`](/actions/vector-operations#fetchvectors) | Queries vectors for similarity search |

### HTTP & External APIs
| Action | Description |
|--------|-------------|
| [`http`](/actions/http-requests#http) | Makes HTTP requests to external APIs |

### Authentication & Security
| Action | Description |
|--------|-------------|
| [`authenticate`](/actions/authentication#authenticate) | Validates JWT tokens and fetches user data |
| [`jwt`](/actions/authentication#jwt) | Encodes or decodes JWT tokens |
| [`hash`](/actions/authentication#hash) | Hashes values using bcrypt |

### Transformation & Logic
| Action | Description |
|--------|-------------|
| [`javascript`](/actions/transformation#javascript) | Executes custom JavaScript code |
| [`static`](/actions/transformation#static) | Returns a static or computed value |

### AI & Agents
| Action | Description |
|--------|-------------|
| [`agent`](/actions/ai-agents#agent) | Interacts with AI models |

### Flow Control
| Action | Description |
|--------|-------------|
| [`parallel`](/actions/flow-control#parallel) | Executes multiple actions simultaneously |

### Communication
| Action | Description |
|--------|-------------|
| [`email`](/actions/communication#email) | Sends email messages via SMTP |

### System
| Action | Description |
|--------|-------------|
| [`command`](/actions/command#command) | Executes shell commands |

### Memory
| Action | Description |
|--------|-------------|
| [`memory_store`](/actions/memory#memory-store) | Stores data in temporary memory |
| [`memory_fetch`](/actions/memory#memory-fetch) | Retrieves data from temporary memory |

### Binance Trading
| Action | Description |
|--------|-------------|
| [`binance/getprice`](/actions/binance#binancegetprice) | Gets cryptocurrency price data |
| [`binance/accountbalance`](/actions/binance#binanceaccountbalance) | Queries Binance account balances |
| [`binance/spotorder`](/actions/binance#binancespotorder) | Places spot trading orders |
| [`binance/futuresorder`](/actions/binance#binancefuturesorder) | Places futures orders |
| [`binance/pricedifference`](/actions/binance#binancepricedifference) | Calculates price changes |
| [`binance/tradeinfo`](/actions/binance#binancetradeinfo) | Gets order and position info |
