ServFlow offers two ways to build APIs: a visual dashboard interface or YAML configuration files. Both approaches use the same underlying engine and produce the same results — choose the one that fits your workflow.

## Choose Your Path

<CardGroup cols={2}>

<Card title="Dashboard Quickstart" icon="window" href="/quickstart-dashboard">
  Build your first API using the visual web interface. Best for learning ServFlow and rapid prototyping.
  
  **Time:** ~5 minutes
</Card>

<Card title="YAML Quickstart" icon="code" href="/quickstart-yaml">
  Build your first API using YAML configuration files. Best for code-first workflows and CI/CD pipelines.
  
  **Time:** ~5 minutes
</Card>

</CardGroup>

## Which Should I Choose?

| If you... | Start with... |
|-----------|---------------|
| Are new to ServFlow | [Dashboard Quickstart](/quickstart-dashboard) |
| Prefer visual interfaces | [Dashboard Quickstart](/quickstart-dashboard) |
| Want to learn how workflows connect | [Dashboard Quickstart](/quickstart-dashboard) |
| Prefer writing code/config files | [YAML Quickstart](/quickstart-yaml) |
| Plan to use version control for APIs | [YAML Quickstart](/quickstart-yaml) |
| Are setting up CI/CD pipelines | [YAML Quickstart](/quickstart-yaml) |

<Tip>
You can switch between approaches at any time. The dashboard saves configurations as YAML files, so you can start visually and later manage configs in code — or vice versa. See [Running Modes](/running-modes) for more details.
</Tip>

## Prerequisites

Before starting either quickstart, make sure you have ServFlow Pro installed:

- **Docker** (recommended) or the ServFlow Pro binary
- A terminal or command-line interface
